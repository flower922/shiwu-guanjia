> **使用说明**：本手册为三位零代码基础的成员（A/B/C）提供逐步操作指令。
> **核心原则：成员不需要在终端输入任何命令。** 所有代码和命令都通过 VSCode 中的 Claude Code 插件完成 —— 你只需用中文描述要做什么，Claude Code 会帮你执行。
> 每个子任务设计得足够小，确保在一次 Claude Code 对话中完成。

> **⚠️ v2 修订说明**（相比初版）：
> 1. **删除了所有需要手动输入的终端命令** —— 成员只需对 Claude Code 说中文
> 2. **Git 统一使用 SSH 连接** —— 不用 HTTPS 地址
> 3. **PR 全流程通过 `gh` 命令完成** —— 不需要打开 GitHub 网页
> 4. **区分了"VSCode Claude Code 插件"和"Claude Code CLI"的职责**
> 5. **补充了操作位置说明** —— 每步都告诉你在哪里做

## 前置知识：两种 Claude Code 的区别

你们会用到两种 Claude Code，做不同的事：

### ① VSCode 里的 Claude Code 插件（三人都用）
- **在哪**：VSCode 左侧边栏的 Claude 图标
- **干什么**：写代码、改代码、执行 git 命令、创建 PR
- **怎么用**：在聊天框里用中文描述你要做什么

### ② Claude Code CLI（只有成员A flower922 用）
- **在哪**：终端里输入 `claude` 启动
- **干什么**：合并 PR（`gh pr merge`）
- **为什么只有A用**：A 是仓库 owner，负责审核和合并所有人的代码

### 通用提醒
- **不要自己打开终端输入命令** —— 让 Claude Code 帮你跑
- **复制本手册方框内的指令，粘贴到 Claude Code 聊天框**
- **如果 Claude Code 问你问题**，用中文回答即可
- **如果上下文快用完**（聊天窗口顶部会有提示），点"新建对话"开一个新的
